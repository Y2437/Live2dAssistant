# Live2dAssistant 项目需求与架构说明

## 1. 文档目的

这份文档用于描述 `Live2dAssistant` 当前版本的产品目标、维护边界、主要能力面与迭代方向。

适用场景：

- 新成员快速理解项目做什么
- 对齐“当前产品目标”与“当前真实维护范围”
- 区分产品目标文档与开发实现文档
- 为后续 roadmap、重构与文档更新提供基线

说明：

- 本文档关注“项目要做什么、当前维护哪些能力、优先级是什么”。
- 具体工程结构请以 `docs/DEVELOPER_ARCHITECTURE_GUIDE.md` 为准。

## 2. 项目定位

`Live2dAssistant` 是一个基于 Electron 的桌面助手项目。当前主交互模式是 `devShell` 单壳大窗口，围绕以下主线持续迭代：

1. 看板娘交互
2. AI 对话与 Agent 工具能力
3. 记忆系统
4. 知识卡片沉淀
5. 番茄钟、日历计划与个人信息管理

当前优先方向依然是大窗口主界面与其核心闭环，而不是扩展更多独立窗口形态。

## 3. 当前产品目标

### 3.1 大窗口主交互

大窗口模式是当前主界面和主要体验入口。

目标体验：

- 左侧保留 Live2D 看板娘区域
- 默认通过浮动气泡显示短回复和触摸反馈
- 当回复较长或 Agent 过程较复杂时，自动扩展为左右分栏
- 右侧承载长对话、执行摘要和过程轨迹
- 用户可以从主界面跳转到其他功能面板

当前状态：

- 已实现大窗口布局与视图切换
- 已实现普通对话与 Agent 流式输出主链路
- 已实现长回复自动展开
- 已实现执行摘要、取消、超时、重试等基础交互

当前限制：

- Agent 执行轨迹仍偏摘要化
- 主界面的结构化任务感仍有提升空间
- renderer 回归验证仍不系统

### 3.2 知识卡片箱

知识卡片箱的定位是结构化知识沉淀容器。

目标要求：

- 以本地 JSON 持久化卡片
- 支持列表展示、搜索、筛选和分页
- 支持用户创建、更新、删除
- 支持摘要生成与详情查看
- 支持与 Agent 的知识沉淀链路协同

当前状态：

- 已支持卡片新建、更新、删除
- 已支持摘要生成
- 已具备基础列表与预览能力

当前限制：

- 卡片沉淀质量和来源追踪仍可继续增强
- 更细的编辑保护和质量校验仍可补齐

### 3.3 AI 记忆系统

记忆系统分为短期上下文与长期记忆两层。

目标要求：

- 聊天上下文本地持久化
- 支持长期记忆提炼与管理
- 提取稳定偏好、事实、习惯与长期任务信息
- 支持手动与自动维护流程

当前状态：

- 已持久化短期上下文
- 已持久化长期记忆
- 已支持手动提炼、删除与 routine 元数据管理
- 已接入每日自动提炼链路

当前限制：

- 去重和污染控制仍有优化空间
- 当前仍不是完整的语义记忆检索系统
- 管理界面与可视化仍偏基础

### 3.4 Agent 能力

Agent 当前已经接入主进程工具链，并作为主交互的一部分持续迭代。

目标方向：

- 能搜索和阅读网页
- 能使用记忆与知识卡片
- 能访问剪贴板与截图能力
- 能读取番茄钟、日历计划等本地结构化信息
- 能在允许的工具范围内完成桌面助手类任务

当前状态：

- 已具备普通模式与 direct mode
- 已支持工具白名单控制
- 已支持流式输出、执行状态与能力自测
- 已接入 memory/cards/clipboard/calendar/todo/AI diary 等一批本地工具能力

当前限制：

- 还不是通用型复杂任务代理
- 多步任务稳定性和失败恢复还需增强
- 部分工具属于“可调用”，但完成质量仍可继续提升

### 3.5 个人信息管理功能面

除 Assistant 外，当前项目还维护以下用户侧功能面：

- 番茄钟
- 剪贴板管理
- 日历计划
- AI 日记
- 设置 / 记忆 / 能力查看 / 模型设置

这些能力的目标不是彼此独立成产品，而是作为桌面助手体系的一部分，共同服务主界面闭环。

## 4. 当前真实维护边界

### 4.1 当前主维护的功能面

当前在代码中明确存在并持续维护的功能面包括：

- Assistant
- Cards
- Pomodoro
- Calendar
- Clipboard
- Settings

### 4.2 当前架构边界

项目采用 Electron 主进程 / preload / renderer 分层：

- 主进程负责业务状态、持久化、模型配置与 Agent 编排
- preload 负责受控 API 暴露
- renderer 负责界面与交互

### 4.3 当前数据边界

当前主要使用本地 JSON / JSONL 文件持久化：

- assistant context
- conversation log
- long-term memory
- model settings
- knowledge cards
- pomodoro
- clipboard history
- calendar plan / todos / AI diaries
- screenshots

## 5. 当前 Agent 真实能力边界（产品视角）

从产品视角看，当前 Agent 能力主要覆盖：

- 对话问答
- 网页搜索与阅读
- 记忆读取 / 提炼 / 管理
- 知识卡片相关操作
- 番茄钟状态读取
- 剪贴板读取与图片分析辅助
- 截图与图像分析
- calendar / todo / AI diary 相关本地数据调用

说明：

- 这里描述的是“产品上已接入的能力范围”，不是承诺每项都已达到高度成熟状态。
- 更细的工具清单与 channel 以开发者架构手册为准。

## 6. 当前已完成的关键成果

- 大窗口主交互链路可用
- Agent 主对话与流式主路径可用
- 记忆系统基础链路可用
- 知识卡片基础链路可用
- 番茄钟、剪贴板、日历计划已进入当前主项目结构
- 模型提供商设置能力已进入系统主链路

## 7. 当前未完成项

### 7.1 高优先级

- Agent 执行过程展示继续结构化
- 多步任务稳定性与失败恢复增强
- 记忆去重与污染控制增强
- 主链路最小回归验证继续补齐

### 7.2 中优先级

- 卡片沉淀质量与保护机制增强
- 日历 / AI diary 的体验打磨
- 剪贴板管理体验增强
- 设置页与模型设置体验继续细化

## 8. 当前代码维护原则

- 一切主体验优先围绕 `devShell` 大窗口模式迭代
- 结构真相以代码与开发者架构手册为准
- main 负责状态与持久化，renderer 不越层直接操作系统资源
- Agent 能力优先服务桌面助手主闭环，而非无限扩张工具面
- 新功能优先补齐闭环与稳定性，不盲目增加入口
