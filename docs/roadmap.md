# Live2dAssistant 迭代 Roadmap

## 1. 文档目的

这份 roadmap 用于把当前项目方向整理成可执行的阶段计划。

它服务于三件事：
- 给后续迭代提供统一优先级
- 避免同时推进过多分散功能
- 为需求拆分、开发排期和验收提供依据

## 2. 当前判断

项目已经完成第一轮可用性搭建：
- 大窗口模式主交互链可用
- Agent 工具链主路径可用
- 记忆系统和知识卡片都有基础链路

当前主矛盾不是“缺少更多入口”，而是以下能力仍不够完整：
- 回复呈现还可以更顺
- Agent 执行状态展示还不够结构化
- 多步任务稳定性仍不足
- 记忆和卡片还没有形成高质量闭环

## 3. 迭代原则

### 3.1 主界面优先

所有直接影响主体验的能力，优先围绕大窗口模式建设，不围绕分窗口扩展。

### 3.2 先打通闭环，再增加功能

优先补齐已有链路中的断点和薄弱点：
- 输出展示
- 执行反馈
- Agent 稳定性
- 记忆沉淀质量

### 3.3 最小测试前置

在流式输出、Agent 编排和关键状态持久化继续重构前，至少建立最小回归测试，防止主链路回退。

## 4. 阶段路线

## Phase 1: 对话体验闭环

### 目标

把“大窗口模式”从“能用”推进到“顺畅可持续使用”。

### 建设重点

- 长回复与短回复的统一渲染策略
- Agent 结构化轨迹面板
- 失败、取消、重试、超时等状态呈现
- 主界面交互细节收口

## Phase 2: Agent 从可调用到可完成

### 目标

让 Agent 不只是“会调工具”，而是能更稳定地完成一段多步任务。

### 建设重点

- 任务拆解
- 步骤执行状态机
- 工具调用失败恢复
- 搜索结果深读
- 执行过程结构化记录

## Phase 3: 记忆与知识闭环

### 目标

把上下文、长期记忆、知识卡片连接成更稳定的知识沉淀系统。

### 建设重点

- 记忆去重
- 污染控制
- 记忆分类质量提升
- 卡片未保存保护
- 卡片来源追踪

## 5. 优先级建议

建议按以下顺序推进：

1. Phase 1 对话体验闭环
2. Phase 2 Agent 从可调用到可完成
3. Phase 3 记忆与知识闭环

## 6. 近期执行顺序

1. 执行摘要面板增强
2. Agent 多步执行状态机与失败恢复
3. 记忆与卡片的数据质量增强
4. 最小回归测试补齐
